# lazyassets Architecture

This document proposes the next `golazy.dev/lazyassets` package. It is a review
artifact, not an implementation contract.

## Goals

- Serve every application static file from the configured public asset sources.
- Generate content-hashed permanent URLs for cache busting.
- Serve permanent URLs with non-expiring cache policy.
- Let framework packages, application packages, and future packages such as
  `lazyjs` register generated assets without writing files to disk.
- Expose helpers that `lazyview` can install into templates.
- Provide stylesheet and JavaScript tag helpers without making `lazyview` know
  asset internals.
- Support minification and precompression.
- Skip large assets by default above 5 MiB, with a configurable policy.
- Let a production app unpack its asset bundle for static hosting or CDN upload.

Non-goals for the first implementation:

- A JavaScript bundler.
- A CSS compiler.
- File watching or hot reload.
- CDN upload.
- Source maps generated by external compilers.

## Inspiration From The Previous Package

The old package at `/home/guillermo/GoLazy/golazy.dev/lazyassets` had the
right broad shape:

- `Storage.AddFS(fs.FS)` mounted filesystem assets.
- `Storage.AddFile(path, content)` registered generated in-memory files.
- `Server` served assets before falling through to the next handler.
- `File.Permalink()` returned a path shaped like `/style-<hash>.css`.
- `ETag` was based on SHA-256 content.
- Permanent URLs received long cache headers.
- CSS `url(...)` references could be rewritten to asset permalinks.
- `lazyapp` exposed a `permalink` helper into views.
- `lazyview/head` had `Style` and `Script` components with useful HTML
  attributes, but they depended on the old `lazyml` package.

Keep those ideas. Do not copy these old details:

- Hashed path parsing was split between `Storage.Find`, `Server.find`, and
  `Server.findWithPrefix`. The new package should resolve logical and permanent
  paths through one manifest.
- Cache headers were only `public, max-age=31536000`. Permanent URLs should use
  `public, max-age=31536000, immutable` by default.
- Conditional `304` responses did not consistently preserve cache headers.
- CSS URL rewriting used a regex-only parser. The first version may still use a
  small scanner, but it must deliberately skip absolute URLs, data URLs,
  fragments, and unresolved paths.
- The old `Stylesheet` type was commented out. The new tag helpers should be
  concrete public API.
- The old script/style components should inspire attribute support, but the new
  implementation should return `lazyview`-compatible fragments instead of
  reintroducing `lazyml`.

## Package Boundary

`lazyassets` should own:

- Asset registration from `fs.FS` and generated content.
- Asset metadata and manifest construction.
- Hashing, ETag, integrity, and permanent URL generation.
- Request handling for static assets.
- Cache policy for logical versus permanent URLs.
- Asset transforms that are safe framework features: minify, CSS URL rewrite,
  and precompression.
- Unpack/export of final asset bytes and manifest metadata.
- View helper functions for asset URLs and common HTML tags.

`lazydispatch` should continue to own generic middleware composition and final
404/405 behavior. Its current `Public` middleware can either become a thin
compatibility wrapper around `lazyassets` or stay temporarily until
`lazyapp.Config.Public` is migrated.

`lazyview` should not depend on `lazyassets`. `lazyapp` wires helpers from
`lazyassets` into the renderer when both packages are present.

`lazy` CLI should own user-facing wrappers such as `lazy assets`, but the
actual export logic belongs in `lazyassets`.

## Core Types

API sketch:

```go
package lazyassets

type Registry struct {
    // setup-time mutable, frozen before serving
}

type Options struct {
    URLPrefix       string        // default "/"
    MaxAssetSize    int64         // default 5 << 20
    OversizePolicy  OversizePolicy
    HashLength      int           // default 12
    LogicalCache    CachePolicy
    PermanentCache  CachePolicy
    Minify          MinifyPolicy
    Compress        CompressPolicy
    RewriteCSSURLs  bool
}

type Asset struct {
    Path        string
    Permanent   string
    ContentType string
    Size        int64
    Hash        string
    ETag        string
    Integrity   string
    Source      string
    Generated   bool
    Ignored     bool
}

type Source interface {
    Assets(*Registry) error
}
```

Registration API:

```go
func New(options ...Option) *Registry
func (r *Registry) AddFS(files fs.FS, options ...SourceOption) error
func (r *Registry) Add(path string, content []byte, options ...AssetOption) error
func (r *Registry) AddReader(path string, open OpenFunc, options ...AssetOption) error
func (r *Registry) Manifest() Manifest
func (r *Registry) ServeHTTP(http.ResponseWriter, *http.Request)
func (r *Registry) Handler(next http.Handler) http.Handler
func (r *Registry) Helpers() map[string]any
func (r *Registry) Unpack(dir string, options ...UnpackOption) error
```

The registry is mutable only during application boot. `lazyapp.New` freezes it
before installing helpers and before serving. Any late registration after freeze
returns an error or panics; the implementation should pick one behavior and test
it explicitly.

## Manifest Model

The manifest is the source of truth:

```text
logical path   -> asset record
permanent path -> same asset record
```

Logical paths are normalized to URL paths:

- They always start with `/`.
- They are cleaned with `path.Clean`.
- They reject `..`, empty paths, invalid UTF-8, and directories.
- They use slash separators independent of the host OS.

Permanent paths use the old user-friendly shape by default:

```text
/css/app.css       -> /css/app-2f6c9de04a1b.css
/favicon.ico       -> /favicon-7e2d94d01cec.ico
/data/schema       -> /data/schema-a0e41f2f6152
```

Do not infer permanent routes by parsing arbitrary request paths at serve time.
Build the permanent-path map when the manifest is built. That avoids ambiguity
for filenames that naturally contain dashes.

Collision rules:

- Duplicate logical paths are errors by default.
- `Replace` source options can deliberately allow later sources to override
  earlier sources.
- Generated assets should be able to use a namespace such as `/assets/lazyjs/`
  without special behavior in the registry.

## Build Flow

Manifest build should be deterministic:

1. Register sources from application public files and generated assets.
2. Enumerate source files in lexical order.
3. Apply oversize policy before content is stored in memory.
4. Load/process small files into asset records.
5. Minify eligible CSS and JavaScript when enabled.
6. Resolve CSS `url(...)` references to permanent URLs when enabled.
7. Hash the final uncompressed served bytes.
8. Precompress final bytes when enabled.
9. Freeze logical and permanent path indexes.

CSS rewriting needs target asset hashes. The implementation should use a
two-pass resolver:

- First pass builds final content and hashes for assets that do not depend on
  URL rewriting.
- Second pass rewrites CSS references using the target asset permanent URLs,
  then hashes the rewritten CSS.

If CSS imports or CSS-to-CSS references create dependency cycles, the first
version should fail with a clear error rather than silently producing unstable
URLs.

## Serving Flow

`lazyapp` should eventually build this chain:

```text
application middleware
router middleware
asset middleware
404 final handler
```

That preserves the current rule that explicit application routes win and static
files are the final root fallback.

Request behavior:

- Serve only `GET` and `HEAD` for existing assets.
- Return `405 Method Not Allowed` with `Allow: GET, HEAD` for unsupported
  methods on existing assets.
- Fall through to the next handler for missing assets.
- Serve logical URLs with revalidation-friendly cache headers and ETags.
- Serve permanent URLs with immutable cache headers.
- Honor `If-None-Match` for both logical and permanent URLs.
- Select precompressed variants with `Accept-Encoding` when available.
- Set `Vary: Accept-Encoding` whenever compressed variants can be served.
- Never precompress or transform responses per request unless explicitly
  configured for development.

Default cache policies:

```text
logical:   public, max-age=0, must-revalidate
permanent: public, max-age=31536000, immutable
```

The permanent cache policy should be configurable, but the default should be
non-expiring in practical browser/CDN terms because content changes produce a
new URL.

## Generated Assets

Generated assets use the same registration and manifest path as disk assets:

```go
assets.Add("/assets/app.js", js, lazyassets.ContentType("text/javascript"))
assets.Add("/assets/app.css", css, lazyassets.ContentType("text/css"))
```

Future packages such as `lazyjs` can expose a source:

```go
type Bundle struct {}

func (b *Bundle) Assets(r *lazyassets.Registry) error {
    return r.Add("/assets/lazyjs/app.js", b.Bytes())
}
```

`lazyapp.Config` should grow an asset hook without forcing application packages
to import implementation details:

```go
type Config struct {
    Public func() (fs.FS, error)
    Assets []lazyassets.Source
}
```

`Public` remains the compatibility path. Internally, `lazyapp` creates one
registry, adds `Public`, runs every `Assets` source, freezes the registry, and
then exposes it through context for advanced setup.

## View Helpers

`lazyassets.Registry.Helpers()` should return helpers that `lazyapp` installs
into `lazyview`:

```text
asset_path "/styles.css"          -> "/styles-<hash>.css"
asset_integrity "/styles.css"     -> "sha256-..."
stylesheet_tag "/styles.css"      -> trusted HTML link tag
javascript_tag "/app.js"          -> trusted HTML script tag
```

`permalink` can stay as a compatibility alias for `asset_path`.

Tag helpers should return `lazyview.Fragment` or `template.HTML` through the
view package integration, not raw strings that are escaped in templates. They
should HTML-escape attribute values and include useful attributes:

```html
<link rel="stylesheet" href="/styles-2f6c9de04a1b.css">
<script src="/app-9e1181b216b2.js" defer></script>
```

Possible Go helpers for non-template code:

```go
func (r *Registry) Path(path string) (string, error)
func (r *Registry) MustPath(path string) string
func (r *Registry) Integrity(path string) (string, error)
func Stylesheet(path string, attrs ...Attr) lazyview.Fragment
func JavaScript(path string, attrs ...Attr) lazyview.Fragment
```

Errors should be visible during rendering. Missing assets should return errors
by default instead of silently returning the original path.

## Minification

Minification should be opt-in for the first implementation:

```go
lazyassets.WithMinify(lazyassets.MinifyCSS(), lazyassets.MinifyJS())
```

CSS and JavaScript minifiers are extension points:

```go
type Transformer interface {
    Transform(AssetInput) (AssetOutput, error)
}
```

The default package can include simple safe minifiers only if they are
conservative. If robust JavaScript minification requires a dependency, keep it
behind a subpackage or leave it to `lazyjs`.

In development, "live minify" can mean transform on each request or first
request after process start. In production, minification should happen at
manifest build time so permanent URLs represent the final served bytes.

## Precompression

Precompression should generate sibling representations for the same asset:

```text
asset bytes
asset bytes + gzip
asset bytes + br
```

Serving selects the best representation from `Accept-Encoding`. The hash,
permanent URL, ETag, and integrity are based on the final uncompressed content.
Compressed variants get the same logical asset identity with
`Content-Encoding` and `Vary: Accept-Encoding`.

Default compression policy:

- gzip enabled for text-like assets when configured.
- brotli behind an optional dependency or subpackage.
- Skip already-compressed formats such as png, jpg, webp, avif, gif, zip, pdf,
  woff, woff2, and gz.
- Skip files below a configurable minimum size.

## Large Asset Policy

Default max processable size: `5 << 20` bytes.

The default policy should be:

- Serve oversized files by logical path if they come from disk.
- Do not hash, minify, rewrite, precompress, or load oversized files fully into
  memory.
- Do not produce permanent URLs for oversized files.
- Return a clear error if a generated in-memory asset exceeds the limit unless
  the caller explicitly allows it.

Configurable policies:

```text
IgnoreForPipeline  default, serve logical only
Error              fail application boot
Allow              process anyway
SkipServing        exclude completely
```

This keeps the first release safe for videos, PDFs, and accidental large binary
files while still serving ordinary static content.

## Unpack Command

The production app should expose asset unpacking through a command path rather
than a long-lived HTTP endpoint. This needs to work from a built binary so a
deployment system can extract exactly the assets embedded in that artifact.

Primary shape:

```sh
./myapp assets unpack --dir public-build
```

`lazy` can provide a developer convenience wrapper:

```sh
lazy assets unpack --dir public-build
```

Direct binary implementation:

1. `cmd/app/main.go` builds the app with `appinit.App()`.
2. Before starting `http.Server`, it passes `os.Args[1:]` to a small framework
   command handler.
3. `lazyassets` freezes the registry, unpacks final asset bytes to the target
   directory, writes a manifest, and returns `true` so `main` exits.

Sketch:

```go
func main() {
    app := appinit.App()
    if lazyassets.HandleCommand(os.Args[1:], app.Assets()) {
        return
    }
    // start http.Server
}
```

This implies `lazyapp.App` exposes its registry:

```go
func (app *App) Assets() *lazyassets.Registry
```

`lazy assets unpack` should mirror the existing `lazy routes` command where
possible:

1. `lazy` locates the application command with `commands/appcmd`.
2. It runs `go run ./cmd/app assets unpack --dir <dir>` for source workspaces,
   or delegates to a configured built binary when one is provided.

The current sample app command has no subcommand parser, so this is a generated
`cmd/app/main.go` contract change. The command handler should stay narrow:
only framework-owned commands under the `assets` namespace, no general app CLI.

Unpack output:

```text
public-build/
  styles.css
  styles-2f6c9de04a1b.css
  styles-2f6c9de04a1b.css.gz
  app-9e1181b216b2.js
  manifest.json
```

The manifest should include logical path, permanent path, hash, integrity,
content type, size, generated/source metadata, and compressed variants.

Unpack should support modes:

- `logical`: write only logical filenames.
- `permanent`: write only permanent hashed filenames.
- `both`: default; useful for gradual CDN migration.

## Integration Plan

Phase 1: package core

- Add `golazy.dev/lazyassets`.
- Implement registry, FS registration, generated asset registration, hashing,
  manifest lookup, ETag, immutable cache headers, and middleware.
- Add focused tests using `fstest.MapFS`.

Phase 2: lazyapp integration

- Replace `lazydispatch.Public` use with `lazyassets.Registry.Handler`.
- Keep `lazyapp.Config.Public` working.
- Add `lazyapp.Config.Assets`.
- Install `asset_path`, `asset_integrity`, `stylesheet_tag`,
  `javascript_tag`, and compatibility `permalink` helpers.
- Update `sample_app` layout to use `stylesheet_tag` or `asset_path`.

Phase 3: transforms

- Add CSS URL rewriting.
- Add minifier extension points.
- Add gzip precompression.
- Add large-asset policies and tests.

Phase 4: CLI/export

- Add `lazyassets.HandleCommand`.
- Update the sample app `cmd/app/main.go` contract to call it before serving.
- Add `lazy assets` subcommands for `list` and `unpack` as wrappers.

Phase 5: documentation

- Update framework README, sample app README, and website guides.
- Document cache behavior and CDN/static-hosting workflow.

## Required Tests

- FS assets serve by logical path.
- Generated assets serve by logical path.
- Permanent URL includes content hash and serves the same bytes.
- Permanent URL receives immutable cache policy.
- Logical URL receives revalidation cache policy.
- `If-None-Match` returns `304` and preserves relevant headers.
- `HEAD` returns headers without a body.
- Unsupported methods return `405` only for existing assets.
- Missing assets fall through to next handler.
- Duplicate logical paths error unless replacement is enabled.
- CSS URL rewriting rewrites relative application assets and skips external,
  data, fragment, and missing URLs.
- Oversized assets follow each configured policy.
- Precompressed variants are selected by `Accept-Encoding`.
- View helpers return escaped, trusted fragments and missing assets error.
- Unpack writes expected files and manifest entries.

## Test Case Layout

Use curated fixture directories for behavior that depends on real filesystem
shape:

```text
lazyassets/testdata/cases/<case-name>/public/...
```

Each case should be small and named after the behavior it protects, for example
`basic`, `css-url-rewrite`, `oversize`, `duplicates`, or `precompressed`.
Use `fstest.MapFS` only when the test is about a single synthetic condition
such as duplicate registration or oversize thresholds.

## Review Questions

1. Should direct production binaries grow an `assets unpack` command now, or
   should this wait behind package and helper stabilization?
2. Should default permanent URL hash length stay at the old 12 hex characters,
   or should it use a longer value such as 16?
3. Should oversized disk files be served logical-only by default, or should they
   be skipped completely?
4. Should JavaScript minification live in `lazyassets` as a conservative
   built-in, or in a future `lazyjs` package?
5. Should the compatibility helper be named `permalink` indefinitely, or should
   docs move users toward `asset_path`?
